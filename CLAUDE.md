## Architecture

**Astro 6, static output** — no server, no JS framework. Every page is pre-rendered to HTML at build time and deployed as a static site on Render.

### Data flow
All content is hardcoded in component frontmatter (the `---` blocks). There is no CMS, no API, and no external data fetching. To change copy, edit the relevant component directly.

### Key files
- `src/layouts/Layout.astro` — single layout wrapping all pages. Contains the Google Fonts `<link>`, cursor DOM elements, and the shared `<script>` block (cursor animation, scroll-reveal IntersectionObserver, smooth scroll).
- `src/styles/global.css` — CSS custom properties (design tokens), reset, shared utility classes (`.rv`, `.btn-primary`, `.tag`, `.s-label`, `.s-title`), and all `@keyframes`. Scoped component styles live inside each `.astro` file.
- `src/components/SEO.astro` — renders all `<head>` meta (title, description, OG, Twitter Card, geo). Takes `title`, `description`, `canonical` props.
- `src/components/SchemaOrg.astro` — inline JSON-LD with `@graph` containing Person, LocalBusiness, and 3× Service schemas. Edit here when contact details or services change.
- `src/pages/index.astro` — single page; imports and composes all section components.

### Scroll reveal
Elements with class `rv` start invisible (`opacity:0; transform:translateY(28px)`) and become visible (`.in`) when they enter the viewport. The IntersectionObserver in `Layout.astro` handles this. Stagger delays use `rv-d1` through `rv-d4`.

### SEO
- Sitemap auto-generated by `@astrojs/sitemap` at build time → `dist/sitemap-index.xml`
- `public/robots.txt` points to the sitemap
- `render.yaml` sets `Cache-Control` headers: immutable for hashed assets in `/assets/*`, `must-revalidate` for HTML

### Deployment
Render Static Site — build command `npm run build`, publish directory `dist`. Config is in `render.yaml`. Every push to `main` auto-deploys.
